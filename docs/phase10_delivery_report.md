# Phase 10：UI 2.0 与动效系统交付报告

## 目标

本阶段将 Android 客户端从工程调试型界面升级为更接近产品形态的移动端 AI 助手界面，并顺手修复两个体验问题：

- 未连接时不再显示“待命”，改为“未连接”；已连接空闲时才显示“待命”。
- 工具调用浮层支持手动上滑或左滑关闭。

## 实现内容

### 1. 状态文案修正

- `Idle` 从“待命”改为“未连接”。
- `Connected` 从“已连接”改为“待命”。
- 顶部状态、开发者抽屉和主视觉状态都跟随新状态文案。

### 2. Assistant Orb 动画化

将原来的静态小表情替换为状态生命体：

- 未连接：低亮度安静球体。
- 待命：柔和呼吸光晕。
- 聆听中：向外扩散波纹和环绕声点。
- 思考中：旋转光弧。
- 说话中：跟随节奏跳动的波形条。
- 错误：红色闪烁和错误形态。

### 3. 主界面视觉重做

- 主界面背景改为 Material 3 + 科技感渐变。
- 顶栏更轻量，减少 PC Qt 复刻感。
- 主视觉卡片使用圆角、光晕、阴影和渐变背景。
- 调试区域改为“开发者抽屉”。

### 4. 按钮与输入区现代化

- 按住说话按钮增加按压缩放和震动反馈。
- 按钮改为更圆润的 pill 风格。
- 底部控制栏增加渐变、圆角和阴影。
- 文本输入和发送按钮重新排布。

### 5. 工具调用提示优化

- 工具调用卡片继续保持悬浮弹窗模式。
- 执行中保持显示。
- 成功、失败、待确认后约 4.5 秒自动隐藏。
- 新增手动关闭：上滑或左滑可收掉。
- 完整工具调用历史仍保留在设置页。

## 验收建议

1. 未连接状态显示“未连接”，连接成功空闲后显示“待命”。
2. 聆听、思考、说话、错误状态下 Assistant Orb 动效不同。
3. 按住说话按钮有按压缩放和震动反馈。
4. 工具调用时主界面出现悬浮卡片，几秒后自动消失。
5. 工具调用悬浮卡片可上滑或左滑关闭。
6. 设置页 MCP 工具历史仍可查看。
7. 文本输入、发送、打断、手动/自动按钮仍正常。

## 备注

本包不改业务协议、不改 MCP 工具执行逻辑、不新增权限。主要是 UI 与交互层升级。
