# Phase 12A KWS 2025 Encoding Fix

## Why this patch exists

The remote repository already points to the 2025 sherpa-onnx KWS model and includes native JNI libraries. The remaining blocker found in remote inspection is `keywords_xiaozhi.txt`: when generated by Windows PowerShell 5.1, the non-BOM UTF-8 `.ps1` script can be interpreted as the system ANSI code page, corrupting pinyin tone marks and Chinese labels.

The corrupted file looked like mojibake instead of:

```text
x iǎo zh ì @小智
x iǎo zh ī @小知
x iǎo zh ì t óng x ué @小智同学
```

This prevents the 2025 KWS model from correctly loading or matching the XiaoZhi wake keywords.

## Changes

- Replaces `keywords_xiaozhi.txt` with valid UTF-8 content.
- Updates `tools/prepare_sherpa_kws_assets.ps1` so it generates keyword lines from Unicode code points. The script is now ASCII-safe and works under Windows PowerShell 5.1.
- Adds `/.tmp_sherpa_*/` to `.gitignore` so temporary download/extract directories are not accidentally committed.

## After applying

Run:

```powershell
Get-Content ".\app\src\main\assets\sherpa-onnx-kws-zipformer-zh-en-3M-2025-12-20\keywords_xiaozhi.txt"
```

Expected output:

```text
x iǎo zh ì @小智
x iǎo zh ī @小知
x iǎo zh ì t óng x ué @小智同学
```

Then build/run from Android Studio or with Gradle.
